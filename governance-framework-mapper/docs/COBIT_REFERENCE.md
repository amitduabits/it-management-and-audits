# COBIT 2019 Quick Reference Guide

## Overview

COBIT (Control Objectives for Information and Related Technologies) 2019 is a governance and management framework for enterprise IT published by ISACA. It provides a comprehensive set of 40 governance and management objectives organized across 5 domains.

---

## Domain Structure

### EDM — Evaluate, Direct and Monitor (5 objectives)
The governance domain. Focuses on board-level oversight of IT.

| ID | Objective | Focus |
|----|-----------|-------|
| EDM01 | Ensured Governance Framework Setting and Maintenance | Governance structures and principles |
| EDM02 | Ensured Benefits Delivery | Value optimization from IT investments |
| EDM03 | Ensured Risk Optimization | Enterprise IT risk appetite |
| EDM04 | Ensured Resource Optimization | IT capabilities and capacity |
| EDM05 | Ensured Stakeholder Engagement | Transparency and accountability |

### APO — Align, Plan and Organize (14 objectives)
Strategic alignment and planning of IT resources.

| ID | Objective | Focus |
|----|-----------|-------|
| APO01 | Managed IT Management Framework | Operating model and processes |
| APO02 | Managed Strategy | IT strategy and roadmap |
| APO03 | Managed Enterprise Architecture | Architecture standards and blueprints |
| APO04 | Managed Innovation | Technology trends and innovation |
| APO05 | Managed Portfolio | Investment and project portfolio |
| APO06 | Managed Budget and Costs | IT financial management |
| APO07 | Managed Human Resources | IT talent and skills |
| APO08 | Managed Relationships | Business-IT relationships |
| APO09 | Managed Service Agreements | SLAs and service catalogs |
| APO10 | Managed Vendors | Third-party and vendor management |
| APO11 | Managed Quality | Quality assurance and standards |
| APO12 | Managed Risk | IT risk assessment and mitigation |
| APO13 | Managed Security | Information security management |
| APO14 | Managed Data | Data governance and quality |

### BAI — Build, Acquire and Implement (10 objectives)
Building and deploying IT solutions.

| ID | Objective | Focus |
|----|-----------|-------|
| BAI01 | Managed Programs | Program and project delivery |
| BAI02 | Managed Requirements Definition | Requirements analysis |
| BAI03 | Managed Solutions Identification and Build | SDLC and development |
| BAI04 | Managed Availability and Capacity | Performance and scalability |
| BAI05 | Managed Organizational Change | Change adoption and training |
| BAI06 | Managed IT Changes | Change control processes |
| BAI07 | Managed IT Change Acceptance and Transitioning | UAT and go-live |
| BAI08 | Managed Knowledge | Knowledge management and docs |
| BAI09 | Managed Assets | IT asset lifecycle management |
| BAI10 | Managed Configuration | CMDB and configuration baselines |

### DSS — Deliver, Service and Support (6 objectives)
Day-to-day IT service operations.

| ID | Objective | Focus |
|----|-----------|-------|
| DSS01 | Managed Operations | IT operations and monitoring |
| DSS02 | Managed Service Requests and Incidents | Incident management |
| DSS03 | Managed Problems | Root cause analysis |
| DSS04 | Managed Continuity | DR and business continuity |
| DSS05 | Managed Security Services | Security operations |
| DSS06 | Managed Business Process Controls | Data integrity controls |

### MEA — Monitor, Evaluate and Assess (5 objectives)
Performance monitoring and compliance.

| ID | Objective | Focus |
|----|-----------|-------|
| MEA01 | Managed Performance and Conformance Monitoring | KPIs and dashboards |
| MEA02 | Managed System of Internal Control | Internal audits |
| MEA03 | Managed Compliance with External Requirements | Regulatory compliance |
| MEA04 | Managed Assurance | Audit planning |
| MEA05 | Managed External Assurance | Third-party audits and SOC |

---

## Key Concepts

- **Governance vs Management:** Governance (EDM) sets direction; Management (APO, BAI, DSS, MEA) executes
- **Design Factors:** COBIT 2019 introduces 11 design factors to customize the governance system
- **Focus Areas:** Specific topics like cybersecurity, DevOps, or small enterprises can be addressed through focus area guidance
- **Performance Management:** Each objective can be assessed at capability levels 0–5 using the COBIT Performance Management model
